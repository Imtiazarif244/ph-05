Question-1: What are the different ways to select an element in the DOM?

1.getElementById – Selects an element by its unique id.
2.getElementsByClassName – Selects elements by class name (returns a live collection).
3.getElementsByTagName – Selects elements by tag name (returns a live collection).
4.querySelector – Selects the first element that matches a CSS selector.
5.querySelectorAll – Selects all elements that match a CSS selector (returns a static NodeList).
6.getElementsByName – Selects elements by the name attribute (commonly used in forms).

Question-2: What is the difference between innerHTML, innerText, and textContent ?

1.innerHTML: Retrieves or sets the HTML content of an element, including tags and markup. Can be used to insert or modify HTML dynamically. Risk of XSS (Cross-Site Scripting) if inserting user input without sanitization.
2.innerText: Retrieves or sets the visible text inside an element, excluding hidden elements. Takes CSS styles into account (e.g., display: none elements are ignored). Slower than textContent because it processes styles. textContent
3.text-content: Retrieves or sets all text content of an element, including hidden text. Does not parse HTML; treats everything as plain text. More performant than innerText since it doesn't consider styles.
Key Differences: innerHTML includes HTML tags, while innerText and textContent only deal with text. innerText respects CSS visibility, whereas textContent retrieves all text regardless of styles. textContent is the most efficient for purely text-based operations.

Question-3: What is event delegation in the DOM?

Event delegation is a technique in the DOM where a single parent element handles events for multiple child elements, even if they are added dynamically.

Question-4: What is event bubbling in the DOM?

Event bubbling is a fundamental concept in the Document Object Model (DOM) that describes how events propagate through the hierarchy of elements.


The Process:


When an event occurs on a specific element (the "target" element), that event doesn't necessarily stop there. Instead, it "bubbles up" through the DOM tree, meaning it travels upward to the target element's parent, then to its grandparent, and so on, until it reaches the root of the DOM (typically the window object).


If any of those ancestor elements have event listeners attached to them for that particular event, those listeners will be triggered.


In simpler terms:


Imagine dropping a pebble in a pond. The ripples spread outward. Similarly, an event "ripples" upward through the DOM structure.

Question-5: How do you create, add, and remove elements using JavaScript?

Creating Elements:


document.createElement(tagName): This method creates a new HTML element with the specified tagName (e.g., "div", "p", "span").

Adding Elements:


parentNode.appendChild(childNode): Appends a childNode as the last child of a parentNode. parentNode.insertBefore(newNode, referenceNode): Inserts a newNode before a referenceNode within a parentNode. parentNode.append(childNode1, childNode2, ...): Appends one or more Node or string objects to the last child of the parent. parentNode.prepend(childNode1, childNode2, ...): Prepends one or more Node or string objects to the first child of the parent.


Removing Elements:


parentNode.removeChild(childNode): Removes a childNode from its parentNode. childNode.remove(): Removes the childNode from the DOM. This is a more modern and generally preferred method. Key Considerations: When removing an element with removeChild(), you need a reference to the parent node. With remove(), you can call the method directly on the element you want to remove. These methods allow for dynamic updates to web pages, enabling interactive and responsive user experiences. It is also possible to add and remove content by manipulating the 'innerHTML' property of an element. This method however, is generally less performant, and can create security issues if not properly sanitized.
